# タイムライン UI 操作仕様

## 目的

- 人間が迷わず使える、標準的で理解しやすい予定編集 UI を定義する
- チャットで作った予定候補を、視覚的に確認して微調整しやすくする

## 基本方針

- まず見て分かる
- 次に触って意図が伝わる
- 誤操作しても戻しやすい
- 細かい調整は必要なときだけ出す

## 採用する操作原則

- 主表示は 1 日 24 時間の縦タイムラインとする
- 予定ブロックは開始位置と長さで意味が伝わる表現にする
- 新規予定は「空いている時間帯をタップ」で作成できるようにする
- 予定の時間変更は「ドラッグ」で行えるようにする
- 予定の長さ変更は「上下ハンドルのドラッグ」で行えるようにする
- 正確な補正用に、時刻入力 UI も併設する

## 標準操作

### 1. 予定作成

- 空きスロットをタップすると、その時刻を開始時刻として仮予定を作成する
- 仮予定の長さは、最初は短すぎない標準値で入れる
- 初期長さは `30 分` を第一候補とする
- 作成直後に編集シートを開き、開始時刻、終了時刻、タイトルをすぐ直せるようにする

### 2. 予定移動

- 既存の予定ブロック本体をドラッグすると、開始時刻と終了時刻を保ったまま移動する
- ドラッグ中は現在の開始時刻と終了時刻をフローティング表示する
- スナップは現在選択中の粒度に合わせる

### 3. 長さ変更

- 予定ブロックの上端と下端にリサイズハンドルを出す
- 上下どちらの端もドラッグできるようにする
- リサイズ中は変更後の時間を明示する

### 4. 詳細編集

- 予定をタップすると詳細シートまたはサイドパネルを開く
- 詳細編集では次を変更できるようにする
- タイトル
- 開始時刻
- 終了時刻
- メモ

### 5. 粒度変更

- タイムライン上部に粒度切り替えを置く
- 切り替え候補は `5 / 10 / 15 / 30 / 60 分`
- デフォルトは `10 分`
- 粒度変更は表示とドラッグスナップに反映する

## モバイルでの操作方針

- 主体はタップ操作にする
- モバイルでは誤操作防止のため、直接ドラッグだけに依存しない
- 予定タップ後のボトムシートから時間を微調整できるようにする
- 5 分粒度では、ドラッグよりも数値入力やステッパー補正を強めにする

## デスクトップでの操作方針

- ドラッグ操作を主導線にする
- ホバー時に編集 affordance を見せる
- チャットとタイムラインを横並びにし、会話しながら編集できるようにする

## 使いやすさの理由

- 縦タイムラインは「時刻が上から下へ流れる」ので直感に合いやすい
- 予定ブロックの高さで長さが伝わるので、一覧認識が速い
- タップで作成、ドラッグで調整、数値で補正の三層構造にすると初心者と上級者の両方に対応しやすい
- 粒度を固定せず切り替え可能にすることで、粗く見たいときと精密に詰めたいときを両立できる

## 非採用とする操作

- 月間カレンダーを主編集面にする
- 長押しだけで主要操作を成立させる
- すべての編集をモーダル入力だけに寄せる

## 非採用の理由

- 月間カレンダーは時間帯管理に弱い
- 長押し依存は発見しづらい
- モーダル入力だけでは、時間の前後関係や空き時間が見えにくい
