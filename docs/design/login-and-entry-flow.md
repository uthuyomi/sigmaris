# ログインと導線

## 目的
- 初回訪問時の導線を分かりやすくする。
- ログイン前の公開画面と、ログイン後のアプリ画面を明確に分ける。

## 画面構成
- `/`
  - 公開トップページ
  - アプリ概要と利用フローを見せる
  - ログインページへの入口を置く
- `/login`
  - Google ログイン専用ページ
  - 初回権限の説明を表示する
- `/app`
  - ログイン後のアプリ側ホーム
  - チャット、カレンダー、タイムライン、設定への入口を置く

## 保護対象
- `/chat`
- `/calendar`
- `/timeline`
- `/settings`

未ログインでアクセスした場合は `/login` へリダイレクトする。

## 実装
- `src/lib/supabase/auth.ts`
  - 現在ユーザー取得
  - ページ保護
- `src/components/auth-controls.tsx`
  - Google ログイン
  - ログアウト
  - ログイン状態表示
- `src/components/app-shell.tsx`
  - アプリ側の共通レイアウト

## ねらい
- 公開トップは説明に集中させる
- ログインページは権限許可に集中させる
- アプリ本体は実際の操作に集中させる
