# 外部情報取り込みパイプライン

## 目的

- スプレッドシートや画像ファイルから勤務情報を読み取り、予定データへ変換する

## 想定フロー

1. ユーザーがチャットにスプレッドシートURLまたは画像を渡す
2. サーバー側で元データを取得する
3. AI が勤務情報を抽出し、内部イベント形式へ変換する
4. AI がチャット上で候補一覧と曖昧点を返す
5. ユーザーがチャットで修正指示または承認を返す
5. 承認後に Google Calendar または自前カレンダーへ反映する

## 入力ソース

- Google Sheets
- 画像ファイル
- 将来候補: CSV / Excel / PDF

## 内部イベント形式の最低項目

- タイトル
- 開始日時
- 終了日時
- メモ
- ソース種別
- ソースID
- 信頼度または確認状態

## UI 上の必要画面

- チャット画面を取り込み入口として使う
- AI 抽出結果の確認 UI はチャット内メッセージとして返す
- 必要に応じて差分修正パネルを補助表示する
- カレンダー反映結果はチャットとカレンダー画面の両方で確認する

## 現在の実装状況

- チャット画面の補助パネルから `Google Sheets URL` と `画像ファイル` を解析できる
- 解析結果は候補一覧として表示できる
- 候補一覧から `Google Calendar` へ反映できる
- まだチャットスレッド本体の添付メッセージとは完全統合していない

## 注意点

- 画像からの抽出は誤読が起きる前提で扱う
- 予定登録は自動確定ではなく確認後反映を基本にする
- 外部ソースの再同期を考えるなら、元データとの紐付け情報を残す必要がある
- スプレッドシートURLは AI が直接読むのではなく、アプリ側で API 経由取得してから渡す
