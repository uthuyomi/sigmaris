# チャット中心 UI とデザインシステム方針

## 目的

- 予定作成や調整をチャットベースで進められる UI を構築する
- スマホアプリに近い軽快さと視認性を Web アプリ上で再現する

## 前提

- アプリ基盤は `Next.js + TypeScript`
- 認証とユーザーデータ管理は `Supabase`
- UI は PC とスマートフォンの両方を対象にする

## 推奨構成

- スタイリング基盤: `Tailwind CSS`
- コンポーネント基盤: `shadcn/ui`
- 低レベル UI プリミティブ: `Radix Primitives`
- チャット UI 基盤: `assistant-ui`
- 会話ストリーミングや AI 連携: `Vercel AI SDK`
- アニメーション: `Motion`

## この構成を推す理由

- `shadcn/ui` は `Next.js` と相性が良く、コンポーネントを手元のコードとして保持できる
- `Radix Primitives` はアクセシブルな UI パターンの土台として信頼しやすい
- `assistant-ui` は AI チャット向けの React UI レイヤーを提供しており、チャット中心画面の立ち上がりが速い
- `Vercel AI SDK` は Next.js 上でストリーミング中心のチャット体験を組みやすい
- `Motion` を使うと、スマホアプリ寄りの滑らかな遷移やシート演出を入れやすい

## デザイン方針

- 画面全体は「チャット画面が主役」の構成にする
- スマホでは下部ナビゲーション、シート、フルハイトの会話画面を優先する
- デスクトップではチャットを中心にしつつ、右ペインやモーダルで補助情報を出す
- 入力体験はメッセージ欄、候補チップ、クイックアクションを重視する
- 予定表示は単なる日付カードではなく、時間帯が見えるタイムラインを前提にする

## UI パターン案

- ホーム: 直近の予定と会話再開
- チャット: アシスタントと予定調整を進める主画面
- 予定一覧: 作成済み予定の確認と編集
- 設定: アカウント連携、通知、プロフィール

## 予定 UI 方針

- 予定の主表現は 24 時間タイムラインとする
- チャットから生成された予定候補を、タイムライン上で確認と微調整できるようにする
- スマホでは縦スクロール型のデイビューを優先する
- デスクトップではチャットと日別タイムラインを横並びにできる構成を検討する
- 月間カレンダーは補助導線に留め、主操作面にはしない

## 時間粒度の暫定方針

- 時間粒度は UI 上で切り替え可能にする
- 初期 UI のデフォルト粒度は 10 分刻みとする
- より細かい調整が必要な場面では 5 分刻みへ切り替えられるようにする
- 切り替え候補は `5 / 10 / 15 / 30 / 60 分` とする
- タイムライン表示は粒度変更に耐えられるレイアウトで設計する

## スマホアプリ寄せで使いたい部品

- Bottom Tabs
- Drawer / Bottom Sheet
- Card
- Segmented Controls
- Command / Quick Actions
- Toast

## いったん見送る案

- `Tamagui` を最初から採用する案

## 見送る理由

- 今回はまず `Next.js` の Web アプリとして立ち上げる前提が強い
- Web だけで十分表現できる段階では、UI 基盤を増やしすぎると設計が重くなる
- 将来 React Native まで同時展開する段階になってから再評価したほうが合理的
