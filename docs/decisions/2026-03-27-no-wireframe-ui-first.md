# 2026-03-27 ワイヤーフレームを省略して UI 先行で進める方針

## 日付

- 2026-03-27

## テーマ

- ワイヤーフレームを先に作るか、そのまま UI 実装に入るか

## 結論

- 初期段階ではワイヤーフレームを必須にしない
- 実際に触れる UI を先に作り、そこから改善していく

## 理由

- ワイヤーフレームに不慣れな状態で先に作ると、設計より手が止まりやすい
- 今回はチャット中心 UI とタイムライン UI の操作感が重要で、静的な図だけでは評価しにくい
- 実装しながら検証したほうが、使いやすさの判断がしやすい

## 影響範囲

- 画面設計は Markdown の仕様メモと実装結果を見ながら更新する
- UI 実装の節目ごとに、判断や変更点をドキュメントへ残す
- 見た目よりも、まず主要操作が成立することを優先する

## 保留事項

- 実装が進んだ段階で、必要なら簡易モックや画面キャプチャによる整理を行うか
