# 2026-03-28 外部情報取り込みと AI 予定変換方針

## 日付

- 2026-03-28

## テーマ

- スプレッドシートや画像ファイルから勤務情報を取り込み、AI によって予定へ変換するか

## 結論

- 外部情報取り込み機能をアプリの主要機能として扱う
- 入力元はまず `Google Sheets` と `画像ファイル` を対象にする
- 入口は専用フォームではなくチャットメッセージを主導線とする
- AI は取り込んだ情報を内部イベント形式へ正規化し、その後 `Google Calendar` または自前カレンダーへ登録する
- AI が自動解釈した結果は、登録前に必ず確認ステップを挟む

## 理由

- 会社の勤務表や出勤情報は手入力より外部取り込みの価値が高い
- スプレッドシートと画像は現場で使われやすい形式
- AI による解釈は有効だが、誤読や曖昧解釈があるため確認工程が必要

## 実装方針

- `Google Sheets API` で表データを読む
- 画像ファイルはチャット添付後に AI で内容を抽出する
- スプレッドシートは URL をチャットで受け取り、アプリ側で API 取得してから AI へ渡す
- 抽出結果は `開始日時 / 終了日時 / タイトル / メモ / 元ソース` を持つ内部イベント形式へ変換する
- 変換後はチャット内で候補一覧を返し、人が承認してからカレンダーへ反映する

## 保留事項

- OCR を OpenAI の画像理解だけで行くか、専用 OCR を併用するか
- 会社ごとにフォーマット差分をどう吸収するか
- 1ファイルから複数人分を読むケースをどう扱うか
