# 2026-03-27 初期認証プロバイダ選定

## 日付

- 2026-03-27

## テーマ

- 初期リリースでどの認証プロバイダを有効にするか

## 結論

- 初期対応の認証プロバイダは `Google` `GitHub` `Discord` とする
- 認証基盤は `Supabase Auth` を利用する

## 理由

- `Google` は幅広いユーザーにとって利用しやすい
- `GitHub` は開発者や IT リテラシーの高いユーザー層と相性が良い
- `Discord` はコミュニティ連携やライトな参加導線と相性が良い
- 初期段階では対応プロバイダを絞ることで実装と検証を進めやすい

## 影響範囲

- Supabase 側で各 OAuth プロバイダ設定が必要
- ログイン UI は複数プロバイダの選択を前提に設計する
- ユーザー属性や利用傾向を見て、後から `Microsoft` やメールリンク認証の追加を検討する

## 保留事項

- 初期リリース後にどの認証手段の利用率が高いか
- 一般業務ユーザー向けに `Microsoft` を追加するか
- メールリンク認証を補助導線として追加するか
