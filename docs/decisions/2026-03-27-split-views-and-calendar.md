# 2026-03-27 画面分割とカレンダー表示の採用

## 日付

- 2026-03-27

## テーマ

- 1画面に詰め込んだ UI を分割するか
- タイムラインとは別にカレンダー表示を持つか

## 結論

- UI は役割ごとにページ分割する
- 初期ページ構成は `ホーム / チャット / タイムライン / カレンダー / 設定` とする
- タイムラインとは別に月表示のカレンダー画面を用意する

## 理由

- 1画面に情報が集まりすぎると視線移動が増えて判断しづらい
- チャットで候補作成、タイムラインで微調整、カレンダーで俯瞰、という役割分担のほうが自然
- 月表示は時間調整には弱いが、予定密度の把握には有効

## 影響範囲

- App Router でページ単位のルーティングを持つ
- 共通ナビゲーションが必要になる
- 同じ予定データを複数ビューで使い回す前提になる

## 保留事項

- ホーム画面をダッシュボードとしてどこまで育てるか
- カレンダーから日単位の詳細画面へ遷移させるか
