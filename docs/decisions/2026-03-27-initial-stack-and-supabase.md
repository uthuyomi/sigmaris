# 2026-03-27 初期技術選定と Supabase 採用方針

## 日付

- 2026-03-27

## テーマ

- 初期技術スタックの選定
- ユーザーごとのデータ保存基盤として Supabase を採用するか

## 結論

- アプリ基盤は `Next.js + TypeScript` を採用する
- ユーザーごとのデータ保存基盤として `Supabase` を採用する
- 外部 API は `Next.js` のサーバー側から接続する

## 理由

- `Next.js` は画面実装とサーバー側の API 中継を一体で扱いやすい
- `TypeScript` により、画面、API、データ構造の型を揃えやすい
- `Supabase` は PostgreSQL ベースで、初期開発に必要な保存基盤を素早く用意できる
- 必要に応じて `Supabase Auth` を使えるため、認証の追加にもつなげやすい
- ユーザーごとのデータを保持する前提では、単純な外部 API 連携だけより保存基盤を早めに持つ価値が高い

## 影響範囲

- データモデル設計は Supabase 前提で進める
- 環境変数管理が必要になる
- API キーや機密情報はクライアント側へ出さず、サーバー側で扱う
- 認証要件が固まったら `Supabase Auth` の採用有無を追加で判断する

## 保留事項

- 保存するデータの具体的なエンティティ定義
- 認証方式の詳細
- Supabase 上でどこまで業務ロジックを持たせるか
