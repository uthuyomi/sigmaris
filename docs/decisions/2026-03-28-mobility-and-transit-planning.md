# 2026-03-28 移動計画を予定調整の一部として扱う

## 日付
- 2026-03-28

## 決定
- 予定に場所がある場合は、移動時間を別画面に追い出さずタイムライン画面で確認できるようにする。
- 出発地は `自宅 / 現在地 / 手入力` を扱う。
- 初期の移動手段は `電車・バス / 車 / 徒歩` を扱う。
- 推奨出発時刻、到着見込み、所要時間を予定判断の材料として返す。

## 理由
- 人間は予定の開始時刻だけでなく、「何時に出れば間に合うか」で行動するから。
- シフトや外出予定では移動込みで時間が埋まるため、予定だけを並べても判断が足りないから。
- チャットで決めた予定候補をタイムラインに落としたあと、そのまま移動まで見える方が自然だから。

## 実装メモ
- `src/components/mobility-panel.tsx` をタイムライン右カラムに配置する。
- `src/app/api/mobility/plan/route.ts` で Google Maps への中継を行う。
- `src/lib/google/maps.ts` で経路レスポンスを UI 向けに整形する。
