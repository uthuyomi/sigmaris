# 2026-03-27 外部カレンダー連携方針

## 日付

- 2026-03-27

## テーマ

- Google カレンダーや他ツールとどう連携するか

## 結論

- 第一優先は `Google Calendar API` 連携
- 次点で `Microsoft Graph Calendar` 連携
- アプリ内部では「カレンダープロバイダ抽象」を用意して、UI や予定ロジックは共通化する
- 初期段階では Nylas のような統合 API には依存しない

## 理由

- このプロジェクトではまず Google 連携の価値が高い
- Microsoft 365 系も業務利用では重要なので、構造上は最初から足せるようにしたい
- UI はプロバイダ差分を意識せず、内部イベント形式で扱ったほうが保守しやすい
- Nylas は便利だが、追加コストと外部依存が増えるため初期段階では避ける

## 実装方針

- 認可は各プロバイダの OAuth を使う
- アクセストークンとリフレッシュトークンはサーバー側で安全に保持する
- 内部では `listEvents / createEvent / updateEvent / deleteEvent / watchChanges` のような共通インターフェースを持つ
- UI は Supabase のユーザーと接続済みプロバイダ情報を見て、同期対象を選べるようにする

## 保留事項

- Google を先に実装するか、認証と並行してプロバイダ基盤から作るか
- 双方向同期を初期版に含めるか
- Apple / CalDAV 系をどこまで対象にするか
