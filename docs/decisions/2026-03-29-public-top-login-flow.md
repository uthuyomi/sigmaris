# 2026-03-29 公開トップとログイン導線をアプリ本体から分離する

## 日付
- 2026-03-29

## 決定
- `/` は公開トップページとして使う。
- `/login` は Google ログイン専用ページとして使う。
- `/app` をログイン後のアプリ側ホームにする。
- `/chat` `/calendar` `/timeline` `/settings` は未ログイン時に `/login` へリダイレクトする。

## 理由
- 初回訪問者に対して、いきなりアプリ本体を見せるより入口を分けた方が分かりやすい。
- ログインの役割とアプリ利用の役割を分離した方が、認証まわりの導線が整理しやすい。
- 今後トップページに説明、導入文、利用フローを追加しやすくなる。

## 実装メモ
- `src/app/page.tsx` を公開トップにする。
- `src/app/login/page.tsx` を Google ログイン画面にする。
- `src/app/app/page.tsx` をアプリ側ホームにする。
- `src/lib/supabase/auth.ts` で `requireUser` を用意し、各ページ保護に使う。
