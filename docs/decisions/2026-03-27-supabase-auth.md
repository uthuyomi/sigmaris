# 2026-03-27 Supabase Auth 採用方針

## 日付

- 2026-03-27

## テーマ

- アカウントで素早くログインできる認証基盤をどうするか

## 結論

- 認証基盤として `Supabase Auth` を採用する
- 初期方針として、ログイン導線は「素早くログインできること」を優先する

## 理由

- すでにデータ保存基盤として `Supabase` を採用する方針と整合する
- 認証とユーザーデータ管理を同じサービス群で扱えるため構成が単純になる
- `Next.js` と組み合わせやすく、サーバー側でセッションや保護ルートを扱いやすい
- 初期開発で独自認証を組むより、実装速度と保守性の面で有利

## 影響範囲

- ユーザー関連テーブル設計は Auth のユーザーIDを前提に考える
- ログイン後の画面保護や API 保護の設計が必要になる
- ログイン方式はメールリンク、OAuth、パスワードレスなどから別途選定する

## 保留事項

- 初期リリースで採用する具体的なログイン手段
- プロフィール情報をどこまで保持するか
- 権限管理をどの粒度で設計するか
