# 2026-03-27 標準的で使いやすいタイムライン UI 方針

## 日付

- 2026-03-27

## テーマ

- 予定編集 UI をどの操作体系で作るか

## 結論

- 標準的なデイタイムライン UI をベースに採用する
- 主編集面は縦型 24 時間タイムラインとする
- 新規作成はタップ、移動はドラッグ、微調整は数値入力の併用とする

## 理由

- 多くのカレンダーや予約 UI で使われる操作体系で、学習コストが低い
- 時間帯中心の要件と相性が良い
- スマホとデスクトップの両方で破綻しにくい

## 影響範囲

- UI コンポーネントはタイムライン中心に設計する
- モバイルではボトムシート中心、デスクトップではドラッグ中心で最適化する
- チャットで作成した予定候補をタイムラインに反映する導線が必要になる

## 保留事項

- 仮予定の初期長さを 30 分固定にするか、文脈に応じて変えるか
- 予定重複時の表示と編集導線
- ドラッグ時の自動スクロール挙動
