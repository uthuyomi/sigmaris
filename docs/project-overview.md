# プロジェクト概要

## 目的

- シフト管理や関連業務を支援する Web アプリケーションを構築する
- ユーザーごとにデータを安全に保持し、外部 API と連携しながら業務を効率化する

## 現在の状態

- 開発開始日: 2026-03-27
- 現在のフェーズ: 初期設計 / 技術選定
- 優先度の高いテーマ: アプリ基盤、認証、ユーザーデータ保存、API 連携方針の確定

## 初期技術方針

- フロントエンド / BFF: `Next.js`
- 言語: `TypeScript`
- ユーザー単位のデータ保存と認証基盤: `Supabase`
- 外部連携: 各種 API をサーバー側から接続

## 想定アーキテクチャ

- UI は `Next.js` で構築する
- 外部 API の呼び出しは、クライアントから直接ではなくサーバー側で中継する
- ユーザーごとのデータは `Supabase` に保存する
- 認証は `Supabase Auth` を採用する
- ログイン体験は「素早く入れること」を重視して設計する

## スコープ

### 含めるもの

- Web アプリケーションとしての基本画面
- ユーザー単位のデータ保存
- ログイン / 認証基盤
- 外部 API 連携
- 今後の拡張を見据えたドキュメント整備

### 含めないもの

- 初期段階での過剰なマイクロサービス分割
- 要件未確定の大規模分析基盤
- 必要性が見える前の複雑なインフラ最適化

## 関連ドキュメント

- `requirements/`: 要件整理
- `design/`: 設計方針
- `decisions/`: 重要な判断記録
